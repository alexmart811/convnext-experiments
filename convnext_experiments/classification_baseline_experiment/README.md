# Эксперимент 1: Базовая классификация на архитектуре ConvNeXt
### Автор: Балицкая Ксения

## Суть эксперимента
Воспроизведение пайплайна обучения модели convnext_tiny для задачи классификации изображений "с нуля" (без предобученных весов) и её прямое сравнение с классической архитектурой resnet50.\
\
Цель эксперимента — проверить способность обеих архитектур к обучению на ограниченных данных с использованием современного рецепта гиперпараметров (описанного в оригинальной статье A ConvNet for the 2020s).

## Датасеты
В данном эксперименте используются наборы данных малого объема для оценки скорости сходимости сетей:

| Название | Задача | Описание |
|----------|--------|--------|
| [Flowers Dataset](https://www.kaggle.com/datasets/imsparsh/flowers-dataset) | Классификация | Набор данных цветов для быстрой проверки сходимости |
| [Mini-ImageNet](https://www.kaggle.com/datasets/deeptrial/miniimagenet) | Классификация | Уменьшенная версия ImageNet, идеальна для тестирования в условиях ограниченных ресурсов |

(Примечание: датасеты скачиваются автоматически при первом запуске с помощью библиотеки kagglehub)

## Структура проекта

> &nbsp;&nbsp;└── `training.py` —  Циклы train_one_epoch и validate, интеграция tqdm

## Выводы по результатам запуска
В ходе тестирования архитектур на 10 эпохах при небольшом объеме данных и батче 32 было установлено:
  * ResNet-50 быстрее сходится и показывает высокую точность благодаря наличию слоев BatchNorm и малых ядер свертки ($3 \times 3$).
  * ConvNeXt-Tiny, имея большие ядра ($7 \times 7$) и слои LayerNorm, требует большего количества эпох, агрессивных аугментаций (Mixup, Cutmix) и огромных объемов данных для успешного обучения "с нуля".
